# Coisas para me lembrar:

* Classe :  na pratica é como se fosse um molde, por exemplo uma caneta ela tem tampa, ponta cor etc, a classe aplica isso a um objeto 

* Objeto : geralmente algo que vc instancia pela classe  e  manipula com atributos;
um exemplo seria a propria caneta que é moldada pela classe

* Atributo: é oq "identifica o objeto" por exemplo a cor da caneta é um atributo, a ponta é um atributo; ja a variavel é um valor ou texto q varia as vezes pode ser constante que geralmente nao deve ser alterado

* Metodo e função: as funções são um bloco de codigo que é usado para nao haver repetição de codigo, exemplo um calculo matematico basico pode ser uma funcao;
ja os metodos n entendi muito bem mas eu imagino que seja algumas "funções" da linguagem que fazem o codigo se comportar de uma maneira expecifica

* Constructor: Constructor ele recebe por parametro e aplica isso aos atributos da classe;
por exemplo uma pessoa faz uma roupa por encomenda, ela precisa das medidas, cor e outras coisas para aplicar isso a roupa, o constructor faz a mesma coisa ele recebe atributos e aplica isso a classe na hora que for instanciar um objeto

* Getter e Setter:Get e Set ate onde eu saiba sao metodos para checar e alterar dados, mais usados quando os dados sao sensiveis ou devem ser protegidos , o metodo get é responsavel por pegar a informacao do atributo X e exibir  aonde o programador pedir, ja o set ele pode alterar o valor de algum atributo por exemplo um usuario jogador de rpg esta nivel 5 quando ele chegar no 10 os atributos dele sobem tantos pontos

* Encapsulamento: o encapsulamento agrupa os metodos e atributos e existe para a proteção da integridade dos dados por exemplo protected e private que o private é necessario um get ou um set para leitura ou alteração ou o protected q tem que pode ser acessado por uma classe estendida

* Metodos e atributos (Static): pelo que eu acabei de pesquisar, metodos e atributos estaticos pertencem a classe nao a instancia individual, é bom usalos em constantes, contadores ou funções utilitarias que nao necessitam da instancia, por exemplo uma funcao statica seria uma transferencia bancaria que nao necessita necessariamente de uma instancia, apenas de um atributo de outra conta para realizar a transferencia

* Herança: a herança permite criar uma classe que herde as caracteristicas de uma classe pai
por exemplo a classe animal e uma classe filha por exemplo cachorro ou humano a classe filha cachorro pode ter todas as caracteristicas animais da classe pai e a calsse filha pode ter outra funcionalidade que a classe pai nao teria que seria latir por exemplo, a classe filha humano pode ter falar ou raciocinar que a classe animal nao tem

* This e Super: This é geralmente ultilizado no metodo constructor com uma unica classe, é usado para modificar atributos de um objeto expecifico por exemplo  o nome de uma pessoa seria passado por parametro e o constructor usaria o this_nome = nome para receber o atributo e passar para este objeto, Super ele funciona com a heranca para puxar atributos da classe pai , funciona basicamente assim vc chama o super como se fosse uma função e coloca o nome do atributo que quer receber da classe pai com o nome do atributo é claro
